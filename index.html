// faq

the obvious objections.

does it actually fix things or just suggest fixes?

actually fixes. on wordpress, webflow, and shopify we push changes via the official api — meta tags, canonicals, alt text, schema, h-tag structure, redirects, image compression, robots.txt. fixes land as drafts by default. flip a toggle and we auto-ship within rules you set.

what cmss are supported?

wordpress, webflow, shopify today. ghost and framer in beta. for anything else, you get the fix list with copy-paste-ready code blocks. custom stacks: we have a github action that opens prs.

can i cancel anytime?

yes. one click in settings. no email required, no exit interview, no "are you sure" three times. cancel mid-month and you keep access through the period.

will it break my site?

every fix is reversible with one click. we keep a diff log. dangerous changes (redirects, robots.txt) require explicit approval even on auto-ship. we've shipped 400k+ fixes and reverted 0.03% of them.

is this just gpt with extra steps?

no. the crawler is ours. the impact-ranking model is trained on 2 years of rank-correlated fix data. llms write the plain-english explanations and draft content — they don't decide what to fix.
